## Компоненты

### 1. Изолированная база знаний

| Параметр | Значение |
|----------|----------|
| Движок | gbrain 0.10.2 |
| Хранилище | PGLite (встроенный PostgreSQL + pgvector) |
| Путь | `/root/rag-brain/` |
| Поиск | Гибридный (векторный + полнотекстовый) |
| Эмбеддинги | OpenAI text-embedding-3-small |
| Поддерживаемые форматы | docx, xlsx, pdf, txt, md, csv, json |

**Изоляция:** RAG-база полностью отделена от основной базы знаний агента. Документы пользователей не смешиваются с личными заметками и контекстом.

**Word Numbering Resolver:** Собственный парсер Word multilevel list numbering (`numbering.xml`). Решает проблему «пустых ячеек» в таблицах — когда коды процессов, нумерация шагов и другие автонумерованные элементы хранятся в XML нумерации, а не как текст. Без resolver'а теряется критически важная структура документов.

**markitdown (Microsoft):** Используется для конвертации xlsx и pdf. Сохраняет структуру таблиц, листов, форматирование. Отмечен как более надёжный чем самописные парсеры для этих форматов.

### 2. Веб-интерфейс

| Параметр | Значение |
|----------|----------|
| Адрес | `http://<сервер>:8787` |
| Стек | Python Flask + vanilla JS |
| Рендеринг | marked.js (markdown → HTML) |
| Сервис | systemd (`rag-ui.service`) |

**Возможности:**
- 📤 Drag & drop загрузка документов (множественная)
- 🗑 Удаление документов из базы
- 🔍 Поиск с LLM-синтезом ответов
- 📋 Список документов с метаданными
- 🌑 Тёмная тема

### 3. RAG Sub-Agent (OpenClaw)

| Параметр | Значение |
|----------|----------|
| Платформа | OpenClaw Gateway |
| Модель | GLM-5.1 (zai) |
| Тип | Изолированный агент |
| Навык | `skills/rag-agent/SKILL.md` |

**Возможности:**
- Работает через чат-интерфейс OpenClaw
- Самостоятельно вызывает gbrain для поиска
- Конвертирует и импортирует документы
- Отвечает с цитатами из источников

### 4. Мультимодельная интеграция

| Модель | Роль | Провайдер |
|--------|------|-----------|
| **GLM-5.1** | Основной агент + RAG субагент | z.ai |
| **GPT-4o-mini** | LLM-синтез ответов в веб-интерфейсе | ProxyAPI (OpenAI) |
| **text-embedding-3-small** | Векторные эмбеддинги | ProxyAPI (OpenAI) |

Все внешние вызовы идут через **ProxyAPI** — единая точка доступа без VPN.

## Поток данных

### Загрузка документа

```
Пользователь загружает файл (.docx / .xlsx / .pdf / ...)
         │
         ▼
Веб-сервер сохраняет оригинал → /root/rag-brain/uploads/
         │
         ▼
Конвертация в Markdown:
  ├─ .docx → Собственный конвертер (python-docx + Word Numbering Resolver)
  │         Парсит numbering.xml, вычисляет автонумерацию,
  │         обрабатывает русские стили заголовков
  ├─ .xlsx → markitdown (Microsoft) — таблицы, листы, структура
  └─ .pdf  → markitdown (Microsoft) — OCR + извлечение текста
         │
         ▼
Markdown сохраняется → /root/rag-brain/sources/
         │
         ▼
gbrain import + embed — индексация и векторизация
         │
         ▼
PGLite: документ разбит на чанки, проиндексирован
```

### Поиск и ответ

```
Пользователь задаёт вопрос
         │
         ▼
gbrain query — векторный поиск
    → 15 релевантных фрагментов, detail: high
         │
         ▼
grep-fallback — полнотекстовый поиск по ключевым словам
    → топ-3 результата по relevance (hit count)
    → добавляются к контексту для точных совпадений
         │
         ▼
Очистка контекста:
    - удаление score-оценок [0.xxxx]
    - замена slug-имён на названия документов
    - добавление списка документов
         │
         ▼
LLM (GPT-4o-mini):
    system: промпт RAG Knowledge Agent
    user: вопрос + очищенные фрагменты
         │
         ▼
Markdown-ответ → marked.js → HTML-рендеринг
```

### Cron-задачи (поддержка базы)

| Задача | Расписание | Назначение |
|--------|-----------|------------|
| gbrain-sync | каждые 15 мин | Синхронизация + эмбеддинги |
| gbrain-dream | ежедневно 02:00 | Ночная обработка сущностей |
| gbrain-health | понедельник 06:00 | Проверка здоровья базы |

## Безопасность

- **Изоляция баз:** RAG-база и основная база агента — разные PGLite-инстансы
- **Аутентификация:** ProxyAPI токен для LLM, gateway token для OpenClaw
- **Локальное хранение:** Все данные на сервере, без внешних облачных хранилищ
- **Удаление:** Полное удаление документа (оригинал + конвертация + индекс)
